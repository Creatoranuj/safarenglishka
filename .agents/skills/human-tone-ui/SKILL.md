# Human Tone UI

Kill the "generated by an AI" smell from every user-facing string and micro-interaction. This app is Hindi/Hinglish-first with English mixed in — write like a real teacher would speak, not like a chatbot.

## When to trigger

- Writing or reviewing empty states, loading text, toasts, dialog copy, button labels, error messages
- User says: "AI jaisa lag rha hai", "make it feel human", "polish the copy", "koi human hi bola ho aisa lage"
- New feature about to ship — before merge, run the checklist
- Auditing an existing surface for tone (paired with `senior-architect-audit`)

## The 4 pillars

1. **Short, kind, direct.** Say the thing. No preamble ("So basically…"), no closer ("Hope this helps! 🌟").
2. **Verb the action.** "Save note", not "Note". "Delete forever", not "Confirm".
3. **Past-tense confirms, present-tense progress.** "Saved" > "Successfully saved!!". "Notes load ho rahe hain…" > "Loading, please wait…".
4. **Respect the user's time and language.** Match the surface's existing Hinglish register. Don't force English on Hindi flows; don't inject Hindi where the surface is English.

## Copy rules

### Buttons & CTAs

- Verb + object, sentence case. `Save note`, `Delete file`, `Add to library`, `Retry`.
- Never `Click here`, `Submit`, `OK` on primary CTAs. `OK` is fine only on info-only dialogs.
- Destructive = red-tinted + explicit verb: `Delete forever`, `Remove access`, `Cancel enrollment`. Not `Confirm`.
- No emoji on buttons unless it's a domain icon carrying meaning (🎧 for audio-only, 🔒 for locked). Never decorative.

### Toasts (Sonner)

- ≤ 5 words when possible. `Saved`, `Copy ho gaya`, `Bookmark added`, `Note deleted`.
- Success = past tense, no exclamation, no emoji.
- Error = one clause of what broke + one clause of what to do. Not "Something went wrong 😔". Instead: `Save nahi ho paaya — dobara try karo.`
- Loading toast optional; if used, verb-object: `Downloading "Chapter 4"…`. Dismiss on completion, don't stack.
- Never `Woohoo!`, `Yay!`, `Oops!`, `Uh oh!`.

### Empty states

Format: **[icon or gradient tile] → one-line "what's missing" → one-line "how to fix" → one CTA**.

Examples (good):
```
[folder icon]
Yahaan koi PDF nahi hai.
Downloads se koi file drag karo ya + button dabao.
[+ Add PDF]
```

Bad:
```
[Sparkles icon]
Nothing here yet ✨
Start your learning journey by adding your first resource!
```

Never use `Sparkles` as the empty-state mascot. Never `Get started` / `Let's begin` / `Journey`. Never a full paragraph.

### Loading states

- Prefer skeletons that match the final layout (project rule via `ListCardSkeleton`).
- If text is unavoidable: verb + noun in Hinglish. `Notes load ho rahe hain…`, `Video ready ho raha hai…`.
- Never bare `Loading…` or `Please wait…`. Never a spinner alone on a full page.
- Progress bar > percentage-less spinner whenever a byte count is known.

### Error messages

Structure: **[what broke] — [what user can do]**.

Good:
- `Video load nahi ho paaya — internet check karke Retry dabao.`
- `AI service abhi reconnect ho raha hai. 10 second baad Retry dabao. 🙏`

Bad:
- `An unexpected error occurred. Please try again later.`
- `Error: NetworkError when attempting to fetch resource.`
- `Oops! Something went wrong 😔`

Never expose stack traces, error codes, or English-only tech jargon to end users. Log the code, show the human line.

### Dialog titles & descriptions

- Title: 2–5 words, states the decision. `Delete this note?`, `Turn off downloads?`.
- Description: 1 sentence with the concrete consequence. `Note delete hone ke baad wapas nahi mil paayega.`
- Buttons name the outcome, not the action: `Delete forever` + `Cancel`. Not `Yes` + `No`.

## Anti-AI tells — flag hard

Any of these means the surface reads AI-generated. Fix before shipping.

**Copy tells**
- `Empowering / Journey / Unlock / Elevate / Revolutionize / Seamless / Effortless / Transform`
- `Let's get started!`, `Welcome aboard!`, `You're all set 🎉`
- `Amazing`, `Awesome`, `Incredible`, `Fantastic` — all filler
- Sentences that end in `✨` for no reason
- Title Case On Every Word Of A Button
- Chatbot-style greetings on first mount ("Hi there! I'm your assistant…")

**Visual tells** (also enforced by `lovable-design-language`)
- Purple→indigo→blue gradient hero on white
- `lucide-react` `Sparkles` icon as brand/agent mark or empty-state mascot
- Generic stock: mountains, rockets, checkmarks-with-confetti
- Every button filled with brand color (no hierarchy)
- "Powered by AI" badges the user didn't ask for

**Behaviour tells**
- Toast every 400ms during a save flow (noise)
- Modal opens on first mount to explain the app
- Suggested prompts that all start with "Can you…"
- Autofilled placeholder text that reads like a marketing tagline

## Checklist (pre-ship)

Run through the surface and answer each:

- [ ] Every button label is a verb.
- [ ] Every empty state has: what's missing, how to fix, one CTA.
- [ ] Every loading state either shows a matched skeleton or verb+noun text.
- [ ] Every error names what broke AND what to do.
- [ ] Every toast is ≤ 5 words, past tense on success.
- [ ] No `Sparkles` icon anywhere as identity/mascot.
- [ ] No purple gradient hero on white (unless the brand actually is that).
- [ ] No filler adjectives (`amazing`, `awesome`, `seamless`, `effortless`).
- [ ] Hinglish matches the surrounding surface — don't code-switch mid-page.
- [ ] Dialog buttons name the outcome, not `Yes`/`No`.
